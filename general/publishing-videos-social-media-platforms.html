    소셜 미디어 플랫폼에 비디오 게시

    이 주제에서는 브라이트코브 Social을 사용하여 소셜 미디어에 비디오를 게시하는 방법을 배웁니다.
     

    브라이트코브 소셜을 사용하면 비디오 클라우드 계정의 동영상을 YouTube, 페이스북, 트위터에 쉽게 게시할 수 있습니다. 비디오를 게시하려면 먼저 비디오를 게시할 소셜 계정에 대한 게시 대상을 만들어야 합니다. 게시 대상 만들기에 대한 자세한 내용은소셜 미디어 게시 대상 만들기를참조하십시오.

    자동 동기화를 사용하고 Social 모듈을 사용하여 비디오를 수동으로 게시하는 작업은 독립적인 작업입니다. 자동 동기화는 Social 모듈을 사용하여 수동으로 게시한 비디오를 인식하지 못합니다. Auto Sync가 소셜 대상 중 하나에 동영상을 게시하도록 하는 방식으로 동영상에 태그가 지정된 경우 해당 비디오가 이전에 Social 모듈을 사용하여 수동으로 게시되었는지 여부에 관계없이 동영상이 적용됩니다.

    게시할 변환 결정

    소셜 플랫폼에 비디오를 배포하려면 비디오에 MP4 변환 또는 비디오 마스터가 하나 이상 있어야 합니다. 브라이트코브가 제공하는 표준 인제스트 프로파일 중 하나를 사용하여 업로드한 동영상은 기본적으로 마스터를 저장합니다. 마스터 보관에 대한 자세한 내용은아카이브 마스터문서를 참조하십시오.

    소셜 플랫폼에 비디오를 배포할 때 브라이트코브 Social은 개별 소셜 플랫폼에 적용되는 규칙을 충족하는 최고 품질의 변환을 배포하려고 시도합니다. 소셜 플랫폼으로 전송된 변환이 거부되면 다른 변환을 보내려는 시도가 없습니다.

    동영상의 순위를 매기는 규칙은 다음과 같습니다.

    • MP4 자산은 다른 비디오 컨테이너보다 우수합니다. 다른 유형의 비디오 컨테이너를 주문하려고 시도하지 않습니다.
    • 비디오 컨테이너의 품질이 동일하면 인코딩 속도가 더 높은 비디오가 더 나은 것으로 간주됩니다.
    • 인코딩 속도가 동일한 품질이면 해상도가 높은 비디오가 더 좋은 것으로 간주됩니다. 동영상의 모든 변환이 반드시 동일한 종횡비를 갖는 것은 아니기 때문에 각 동영상의 높이와 너비의 곱을 사용하여 해상도에 대해 계산된 '복합' 프록시를 기반으로 비디오를 비교합니다.

    비디오 자산 중 하나에 비교에 필요한 위의 정보가 부족한 경우 해당 동영상은 필요한 모든 정보가 들어 있는 동영상보다 '나쁜' 것으로 간주됩니다.

    소셜 미디어에 동영상 게시

    게시 대상을 만든후에는 다음 단계에 따라 비디오를 소셜 미디어에 게시합니다.

    1. 브라이트코브 소셜 오픈.
    2. 왼쪽 탐색에서 [ 비디오 ] 를 클릭합니다. Video Cloud 계정의 동영상 목록이 표시됩니다. 소셜 미디어 계정에 게시되지 않은 동영상의상태는'게시 안됨'으로표시됩니다. 게시된 동영상은상태 확인을표시합니다. 링크를 클릭하면 소셜 미디어 대상에서 비디오의 게시 상태를 볼 수 있습니다.
    3. 게시할 비디오를 클릭한 다음 게시를 클릭합니다.
      참고: DRM이 활성화된 동영상은 비디오 썸네일에 잠금 아이콘을 표시합니다. 동적 전송 인제스트 프로필을 사용하여 인제스트된 DRM 지원 비디오는 게시할 수 있지만 다운로드하거나 클립을 만들 수는 없습니다. DRM 지원 비디오를 선택하면 이 버튼이 비활성화됩니다.
    4. 비디오를 게시할 대상을 선택합니다. 새 대상 추가를 클릭하여 새 대상을만들수도 있습니다.
      참고: 오류가 있는 대상에는 이름 옆에 오류 아이콘 ( ) 이 표시됩니다. 대상을 선택한 다음다시 연결을클릭하여 문제를 해결합니다.
    5. 각 대상에 대한 비디오 메타데이터 속성을 검토합니다. 동영상 속성은 게시하는 소셜 미디어 플랫폼에 따라 다릅니다. 자세한 내용은 각 소셜 미디어 플랫폼에 대한 섹션을 참조하십시오.
      • 유튜브
      • 페이스 북
      • 트위터
    6. [ 대상에 게시 ] 단추를 클릭하여 비디오를 게시합니다. 왼쪽 탐색에서게시됨을클릭하여 배포 상태를 모니터링합니다. 게시 스트림 상태 보기에 대한 자세한 내용은소셜 미디어 게시 스트림 보기를참조하십시오.

    게시 버튼을 사용하여 비디오를 게시하는 것은 일회성 게시 작업입니다. 이 방법을 사용하여 게시된 비디오는 최초 업로드 후 Social 모듈에서 메타데이터 업데이트를 수신하지 않습니다.

    유튜브에 대한 비디오 속성 설정

    기본적으로 최대 15분 길이의 동영상을 YouTube에 업로드할 수 있습니다. 더 긴 동영상을 업로드하는 방법에 대한 자세한 내용은YouTube에서 더 긴 동영상 사용을참조하십시오. 동영상이 YouTube에 게시되면 스튜디오에서 소스 동영상에 대한 업데이트가 YouTube 측에서 업데이트되지 않습니다.

    YouTube 대상을 선택할 때 다음 비디오 속성을 구성 할 수 있습니다.

    • 썸네일 소스
    • 게시물 정보
    • 일정
    • 캡션
    • 가시성
    • 라이센스
    참고:
    • YouTube의 기본 동영상 속성은대상을 편집하여 설정할 수 있습니다.
    • YouTube API는 현재 API를 통해 댓글을 비활성화하는 것을 지원하지 않으므로 소셜 UI에서 댓글을 비활성화할 수 있는 옵션이 없습니다.
    • 비디오의 이름과 설명에 특수 문자를 사용하지 마십시오. “" 를 사용하면 오류가 발생할 수 있지만 다른 특수 문자도 문제가 발생할 수 있습니다.

    축소판 원본 구성

    YouTube에 게시할 때 동영상 썸네일 소스를 다음과 같이 설정할 수 있습니다.

    • 유튜브 - 유튜브는 미리보기 이미지를 선택합니다
    • 비디오 클라우드 - 비디오 클라우드 썸네일 이미지를 YouTube 썸네일로 사용
    • 사용자 지정 이미지 - 사용할 사용자 지정 축소판 이미지를 찾아 선택합니다.

    모든 대상에 대해 사용을선택하면 현재 게시 중인 모든 대상에 대해 이 축소판이 사용됩니다.

    참고: 브라이트코브 소셜에서 YouTube에 맞춤 동영상 미리보기 이미지를 게시하려면 먼저 YouTube 계정을 확인해야 합니다. YouTube 계정 확인에 대한 자세한 내용은이 링크를참조하십시오.
    참고: Studio에서 썸네일 이미지를 업데이트해도 YouTube 측의 썸네일 이미지는 업데이트되지 않습니다.

    게시물 정보 속성 구성

    POST 정보 정보링크를 클릭합니다. 다음 YouTube 동영상 속성을 설정할 수 있습니다.

    • 제목 - 비디오 제목입니다 (기본값은 비디오 클라우드 제목입니다). YouTube는 100 문자까지 지원합니다.
    • 설명 - 비디오에 대한 설명 (기본값은 Video Cloud 간단한 설명입니다). 유튜브는 5000 문자까지 지원합니다.
    • YouTube 카테고리 - 동영상의 콘텐츠 카테고리
    • 태그 - 비디오의 태그 (기본값은 비디오 클라우드 태그입니다). 특정 태그 수 제한은 없습니다. 태그당 최대 30자를 사용할 수 있으며 텍스트 필드에는 최대 500자를 사용할 수 있습니다.
    태그의 문자 제한에 대해 유의해야 할 몇 가지 사항은 다음과 같습니다.
    1. 속성 값은 목록이며 목록의 항목 사이의 쉼표는 500자 제한에 포함됩니다.
    2. 태그에 공백이 포함된 경우 API 서버는 태그 값을 따옴표로 묶은 것처럼 처리하고 따옴표는 문자 제한에 포함됩니다. 따라서 와 같은 태그가있는late news경우 이는 11 자 (따옴표 포함) 로"late news"처리되고 계산됩니다.
    3. 태그의 문자 제한을 초과하면 Social 모듈에 오류가 보고되지 않습니다. 동영상이 성공적으로 공유되었는지 확인하려면 왼쪽 탐색 메뉴에서게시를클릭하고 YouTube에서 확인하거나소셜 API를사용하여 상태를 확인합니다.

    일람표 특성 구성

    일정 ( Schedule ) 링크를 클릭하고 일정 설정을 선택합니다.

    • 즉시 게시 - 비디오를 즉시 게시하려면
    • 게시 날짜 예약 - 비디오의 게시 날짜 및 시간을 선택합니다.
    참고: 일정 날짜와 시간은 동영상이 YouTube로 전송 될 때 한 번만 YouTube로 전송됩니다. 일정 날짜 및/또는 시간을 변경해야 하는 경우 YouTube에서 변경해야 합니다. Social 모듈은 YouTube에 처음 동기화한 후 일정 날짜나 시간을 업데이트하지 않습니다.
    참고: 가지고비디오 가시성로 설정미등록또는은밀한 , 설정시간표옵션즉시 게시그리고 설정시계옵션미등록또는은밀한 .

    캡션 속성 구성

    캡션링크를 클릭합니다. 캡션의 소스를 선택합니다.

    • 비디오 클라우드 캡션사용 - 비디오와 연결된 캡션을 사용합니다. Video Cloud의 자막 작업에 대한 자세한 내용은개요: 비디오에 캡션 추가
    • 자막 없음 - YouTube에 자막이 전송되지 않습니다
    • 캡션 업로드 - 캡션 업로드를클릭하고 캡션 파일을 찾아 선택합니다. 캡션 파일은.VTT 또는.SRT 형식이어야 합니다. 캡션언어를선택하고레이블을추가한 다음형식을선택합니다. 캡션 파일 추가를클릭합니다.
    자막이 있는 동영상은 Social을 통해 클리핑할 수 있지만 클리핑된 동영상에는 자막이 없습니다. 클리핑된 비디오에 새 자막을 업로드해야 하며 Social에서는 게시할 때 해당 자막을 사용합니다.

    가시성 특성 구성

    가시성링크를 클릭하고 비디오에 대한 개인 정보 설정을 선택합니다. 다음과 같은 개인 정보 설정을 사용할 수 있습니다.

    • 공개 - 누구나 동영상을 볼 수 있음
    • 미등록 - 동영상에 대한 링크가 있는 사용자는 누구나 동영상을 볼 수 있습니다.
    • 비공개 - 동영상은 본인 및 선택한 사용자만 볼 수 있습니다.
    참고: YouTube 목적지를 사용하면 동영상과 연결된 사용자 지정 필드를 사용하여 개인 정보 보호 설정을 동적으로 할당할 수 있습니다. 자세한 내용은비디오의 개인 정보 보호 상태 설정을참조하십시오.

    동영상이 YouTube에 게시될 때 구독자에게 알림설정도 있습니다. 이 옵션을 선택하면 구독자가 구독 피드에서 내 동영상을 볼 수 있으며 동영상이 업로드되었다는 알림을 받을 수 있습니다.

    참고: YouTube 개인 정보 보호 설정에 대한 자세한 내용은YouTube 설명서를 참조하십시오 .

    라이센스 속성 구성

    라이선스링크를 클릭하고 비디오에 대한 라이선스 및 권한 소유권 설정을 선택합니다. 다음 설정을 사용할 수 있습니다.

    • 표준 유튜브 라이선스
    • 크리에이티브 커먼즈 - 저작자표시
    참고: YouTube 라이선스 및 권리 소유권에 대한 자세한 내용은YouTube 설명서를참조하십시오.

    동영상이 Facebook에 게시되면 Studio의 소스 비디오에 대한 업데이트가 Facebook 측에서 업데이트되지 않습니다.

    참고: Facebook의 기본 비디오 속성은대상을 편집하여 설정할 수 있습니다.
    참고: 페이스 북은.gif 이미지를 허용하지 않습니다. 동영상에서 축소판 이미지에.gif를 사용하는 경우 Facebook에 게시하지 못합니다.

    축소판 원본 구성

    Facebook에 게시 할 때 비디오 썸네일 소스를 다음과 같이 설정할 수 있습니다.

    • 페이스 북 - 페이스 북은 미리보기 이미지를 선택합니다
    • 비디오 클라우드 - 비디오 클라우드 썸네일 이미지를 Facebook 썸네일로 사용
    • 사용자 지정 이미지 - 사용할 사용자 지정 축소판 이미지를 찾아 선택합니다.

    모든 대상에 대해 사용을선택하면 현재 게시 중인 모든 대상에 대해 이 축소판이 사용됩니다.

    참고: Studio에서 썸네일 이미지를 업데이트해도 Facebook 측의 썸네일 이미지가 업데이트되지 않습니다.

    게시물 정보 속성 구성

    POST 정보링크를 클릭합니다. 다음과 같은 Facebook 비디오 속성을 설정할 수 있습니다.

    • 제목 - 비디오 제목입니다 (기본값은 비디오 클라우드 제목입니다). 페이스 북은 65 자 후에 잘립니다.
    • 상태 메시지 - 비디오에 대한 설명입니다 (기본값은 Video Cloud 간단한 설명입니다). Facebook은 검색 API를 비활성화했기 때문에 Facebook 언급을 자동으로 완성하는 것은 불가능합니다.
    • 태그 - 목록에서 Facebook 태그를 선택합니다. 페이스 북은 태그를 관리합니다. Social 모듈은태그필드에 입력할 때 Social이 목록을 확인하도록 해당 모듈을 준수해야 합니다.
    • 페이스 북 카테고리 - 비디오에 대한 콘텐츠 카테고리

    일람표 특성 구성

    일정링크를 클릭합니다. [ 즉시 게시 ] 를 선택하여 비디오를 즉시 게시하거나 [ 게시 날짜 예약 ] 을 선택하고 [ 게시 날짜 및 시간 ] 을 선택합니다. 게시물이 만료되지않도록 설정하거나만료 예약을 선택한 다음 만료날짜 및 시간을선택할 수도 있습니다. 시간대를선택할 수도 있습니다.

    참고: 일정 날짜와 시간은 동영상이 페이스 북으로 전송 될 때 한 번만 페이스 북으로 전송됩니다. 일정 날짜 및/또는 시간을 변경해야 할 경우 Facebook에서 변경해야합니다. Social 모듈은 Facebook에 초기 동기화 후 일정 날짜 또는 시간을 업데이트하지 않습니다.

    캡션 속성 구성

    캡션링크를 클릭합니다. 캡션의 소스를 선택합니다.

    • 비디오 클라우드 캡션사용 - 비디오와 연결된 캡션을 사용합니다. Video Cloud의 자막 작업에 대한 자세한 내용은개요: 비디오에 캡션 추가

      캡션의 경우 Facebook에는language , 소셜이language텍스트 트랙에name , 소셜이label텍스트 트랙에서.

      name는 '설정' 메뉴와 '선택 자막' 메뉴를 통해 Facebook 플레이어의 시청자에게 표시됩니다. 텍스트 트랙 레이블은 시청자에게 의미 있는 것으로 설정하는 것이 좋습니다. (예를 들어, 해당 언어의 다른 언어로 캡션에 레이블을 지정합니다.)

    • 자막 없음 - YouTube에 자막이 전송되지 않습니다
    • 캡션 업로드 - 캡션 업로드를클릭하고 캡션 파일을 찾아 선택합니다. 캡션 파일은.VTT 또는.SRT 형식이어야 합니다. 캡션언어및형식을선택합니다. 캡션 파일 추가를클릭합니다.
    참고: Facebook에서는 자막 언어에 언어 외에 국가가 포함되어야 합니다 (예: en-US 또는 en-uk). 동기화 할 때 국가 코드 (예: 'us') 의 언어 코드가 실패합니다.
    자막이 있는 동영상은 Social을 통해 클리핑할 수 있지만 클리핑된 동영상에는 자막이 없습니다. 클리핑된 비디오에 새 자막을 업로드해야 하며 Social에서는 게시할 때 해당 자막을 사용합니다.

    대상 속성 구성

    대상링크를 클릭하여 비디오를 볼 수 있는 위치를 설정합니다. 다음 옵션을 사용할 수 있습니다.

    • 청중 최적화 관심사 - 게시물을 즐길 가능성이 가장 높은 사람들의 관심사를 설명하는 태그를 추가합니다. 자세한 내용은Facebook 설명서를 참조하십시오 .
    • 위치 제한 - 게시물에 포함하거나 제외할 국가 추가

    수익 창출 속성 구성

    수익 창출링크를 클릭합니다. 다음 속성을 설정할 수 있습니다.

    • 이 목적지에 게시물에 대한 귀하의 비즈니스 파트너는 누구입니까? - 브랜드 콘텐츠 게시물에 대한 비즈니스 파트너를 추가합니다. 입력하면 일치하는 비즈니스 목록이 나타납니다. 페이스북 브랜드 콘텐츠에 대한 자세한 내용은페이스북 설명서를 참조하십시오 .
    • 비즈니스 파트너가 게시물을 높일 수 있도록 허용

    권한 속성 구성

    참고: 권한 옵션을 표시하려면 Facebook 계정에 Facebook 권한 관리자에 대한 액세스 권한이 있어야 합니다.

    권리링크를 클릭합니다. 다음 속성을 설정할 수 있습니다.

    • 일치하는 콘텐츠 검색 - Facebook 권한 관리를 활성화하려면 선택합니다. 자세한 내용은Facebook 설명서를 참조하십시오 .
    • 소유권 지정 - 비디오, 오디오 또는 둘 다에 대한 저작권 소유권 지정
    • 국가 선택 - 전 세계 , 국가 포함 또는 국가제외를선택합니다. 포함 또는 제외를 선택한 경우위치 제한을추가합니다.
    • 일치 규칙 적용 - 규칙을 선택합니다. 일치 규칙은사람들이 일치하는 동영상을 업로드할 때 발생하는 자동 동작을 결정합니다. 경기 규칙은 페이스 북의 내부에서 관리됩니다.
    • 권한 부여 - 특정 사람이나 페이지 또는 Instagram 계정에 귀하의 콘텐츠와 일치하는 동영상을 게시 할 수있는 권한을 부여합니다.

    배포 속성 구성

    Distribution링크를 클릭하여 비디오를 볼 수 있는 위치를 설정합니다. 다음과 같은사용자 지정 배포 및 중재옵션을 사용할 수 있습니다.

    • 뉴스 피드에 표시 - 뉴스 피드에 비디오 표시
    • 임베딩허용 - 타사 사이트에 비디오 임베딩 허용
    • 비밀 동영상으로 추가 - 동영상은 직접 URL을 통해서만 액세스할 수 있으며 Facebook에서 검색할 수 없습니다.
      • 반응, 댓글 및 공유 끄기 - 좋아요 및 댓글 허용 안 함

    • 게시물 정보

    참고: Twitter의 기본 동영상 속성은대상을 편집하여 설정할 수 있습니다.

    게시물 정보 속성 구성

    POST 정보링크를 클릭하고트윗콘텐츠를 입력합니다. @mentions 트위터로 전송될 때 #hashtags 로 변환됩니다. 자세한 내용은 블로그 게시물인브라이트코브 소셜에서 트위터 @mentions 사용 중단을참조하십시오.

    클립 만들기

    브라이트코브 소셜은 Video Cloud 계정에 있는 기존 비디오에서 비디오 클립을 만들 수 있는 기능을 제공합니다. 클립은 소셜 미디어에 게시할 수 있는 짧은 티저 비디오로 사용할 수 있습니다. 클립을 만들려면 [ 클립 만들기 ] 를 클릭합니다. 클립 만들기에 대한 자세한 내용은브라이트코브 소셜을 사용하여 비디오 클립 만들기를 참조하십시오.

    비디오 다운로드

    브라이트코브 소셜은 비디오 URL을 복사하거나 비디오 소스 파일을 다운로드할 수 있는 기능을 제공합니다. 다운로드 시 최고 품질의 MP4 변환이 다운로드됩니다. MP4 변환이 없으면다운로드링크가 비활성화됩니다.

    참고: DRM이 활성화된 비디오는 다운로드할 수 없습니다 (잠금이 비디오 축소판에 나타남). DRM 지원 비디오를 선택하면다운로드버튼이 비활성화됩니다.

    다음 단계에 따라 비디오를 다운로드합니다.

    1. 브라이트코브 소셜 오픈.
    2. 왼쪽 탐색에서 [ 비디오 ] 를 클릭합니다. Video Cloud 계정의 동영상 목록이 표시됩니다.
    3. 다운로드할 비디오를 클릭한 다음다운로드를클릭합니다.
    4. 동영상 URL을 복사하거나 링크를 클릭하여 브라우저에서 비디오를 엽니다. 비디오를 마우스 오른쪽 단추로 클릭하여 소스 파일을 로컬에 저장합니다.
    5. 완료를 클릭합니다.

    다운로드 링크의 목적은 파일을 다운로드하거나 URL을 소셜 플랫폼의 업로더 모듈 (URL에서 업로드를 지원하는 플랫폼의 경우) 에 제공할 수 있는 값으로 사용하는 것입니다. URL은 공용 소셜 플랫폼 게시물에 붙여 넣을 수 있는 수명이 길고 공개적으로 액세스할 수 있는 URL이 아닙니다.

    미디어 모듈에서 비디오 보기

    비디오 페이지의 비디오는 미디어 모듈에서 비디오를 클릭한 다음미디어에서 보기를클릭하여 볼 수 있습니다.

    게시된 위치 보기

    소셜 미디어에 게시하려고 시도한 동영상은상태 열에 상태 확인이표시됩니다.

    상태 확인링크를 클릭하면 비디오가 게시된 소셜 대상 목록과 게시 상태가 표시됩니다.

    360° 비디오 게시

    유튜브, 페이스북, 브라이트코브 플레이어 모두 360° 비디오 재생을 지원합니다. 브라이트코브 소셜을 사용하여 360° 동영상을 YouTube 또는 Facebook에 게시하면 동영상이 360° 모드로 재생됩니다. 360° 효과가 처리되는 동안 지연이 발생할 수 있습니다. Video Cloud에 360° 비디오를 업로드하기 전에 몇 가지 작업을 수행해야 합니다. 360° 비디오 작업에 대한 자세한 내용은360° 비디오 작업을참조하십시오. 동영상이 업로드되면 360° 동영상을 게시하는 과정은 일반 동영상과 동일합니다.

    참고: 360° 비디오 재생은 특정 브라우저로만 제한됩니다.
    참고: Social 모듈을 사용하여 360° 비디오를 동기화하는 경우 비디오를 인제스트하는 데 사용되는 인제스트 프로파일을 마스터 파일을 저장하도록 구성해야 합니다. 그렇지 않으면 비디오가 360에서 동기화되지 않습니다.
